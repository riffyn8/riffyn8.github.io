## 클래스와 객체
- **클래스**: 객체를 정의해 놓은 '설계도' 또는 '틀'. 객체를 생성하는 데 사용한다.
- **객체**: 설계도(클래스)를 바탕으로 실제로 존재하는 사물이나 개념으로, 속성과 기능의 집합체라고 할 수 있다.
- **인스턴스**: 클래스로부터 객체를 만드는 과정은 인스턴스화라고 하며, 어떤 클래스로부터 만들어진 객체를 그 클래스의 **인스턴스(Instance)**라고 부른다.

> - 객체: 모든 인스턴스를 통칭하는 포괄적인 의미
> - 인스턴스: 해당 객체가 '어떤 클래스로부터 생성되었는지' 그 관계에 초점을 맞춘 의미

인스턴스는 `new` 연산자를 통해 생성할 수 있으며, 생성된 인스턴스는 참조변수를 통해서만 접근할 수 있다.
참조변수는 하나의 인스턴스만 가리킬 수 있다. 이는 참조변수가 가리키는 주소값은 하나이기 때문에 여러 인스턴스를 동시에 가리키는 것이 불가능하기 때문이다.

> 자바는 항상 Call by Value 방식을 사용하는데, 이는 메서드에 인자로 전달되는 값이 복사되어 전달된다는 것을 뜻한다.
> 기본형(Primitive Type)은 값 자체가 복사되어 전달되고, 참조형(Reference Type)은 인스턴스의 주소값이 복사되어 전달된다.
> 따라서 참조형 변수를 통해 메서드 내에서 객체의 필드를 수정하면 원본 객체에도 영향을 미친다. <br/>
> 인스턴스를 참조변수로 접근하는 이유도 이와 관련이 있다. 인스턴스 그 자체의 데이터를 직접 저장하고 이동시키다보면 메모리 낭비가 심해지기 때문에,
> heap 영역에 있는 인스턴스의 주소만을 stack 영역의 참조변수가 가지고 이를 통해 접근하는 방식을 사용하는 것이다.

## 변수와 메서드
변수와 메서드는 선언된 위치와 키워드에 따라 종류와 생명주기가 달라진다.

`static` 키워드를 사용하여 선언된 변수와 메서드를 **클래스 변수**와 **클래스 메서드**라고 부른다. 
이들은 클래스가 메모리에 로드될 때 생성되므로 `new` 연산자로 인스턴스를 생성하지 않아도 사용할 수 있다.
클래스 변수는 클래스가 메모리에 유지되는 동안 모든 인스턴스가 동일한 값을 공유하며, 인스턴스가 여러 개 생성되더라도 값은 하나로 유지된다.
반면 클래스 메서드 내에서는 인스턴스 변수와 인스턴스 메서드를 사용할 수 없다. 
클래스가 로드되는 시점에는 인스턴스가 아직 생성되지 않았을 수 있어, 인스턴스 멤버가 메모리에 올라와 있다는 보장이 없기 때문이다. <br/> 

`static` 키워드 없이 클래스 내부에 선언된 변수와 메서드는 **인스턴스 변수**와 **인스턴스 메서드**라고 부른다.
이들은 `new` 연산자로 인스턴스를 생성한 후, 해당 인스턴스를 통해 접근하여 사용할 수 있다. <br/>

지역 변수는 메서드, 생성자, 초기화 블럭 내부에서 선언되는 변수로, stack에 잠시 머물렀다가 해당 블럭이 종료되면 사라진다.
따라서 선언된 블럭 내에서만 사용할 수 있으며, 
메서드 내에서 연산된 값을 외부로 가져오기 위해서는 반환값을 사용하거나 참조형을 매개변수로 받는 방식을 활용해야 한다.

> 인스턴스 변수와 클래스 변수를 통틀어서 멤버 변수라고 부른다.

## 생성자와 초기화 블럭
**생성자(Constructor)**는 인스턴스가 생성될 때 호출되는 초기화 메서드로, 클래스 이름과 동일해야 하며 반환 타입을 선언하지 않는다.
`void`조차 붙이지 않으며, 붙이면 생성자가 아닌 일반 메서드로 인식된다. 
단, 생성자는 인스턴스 변수들의 초기화를 도와주는 역할을 할 뿐이며, 객체를 직접 생성하는 것은 `new` 연산자의 역할이다. <br/>

생성자 외에도 초기화 블럭을 통해 변수를 초기화할 수 있다. 
인스턴스 초기화 블럭은 생성자보다 먼저 실행되므로 인스턴스에 공통적으로 적용할 초기화 코드를 작성하기에 적합하고,
`static` 키워드를 사용하는 클래스 초기화 블럭은 클래스가 메모리에 처음 로딩될 때 단 한 번 실행되며 클래스 변수를 초기화하는 데 사용된다.

> 실행 순서: 클래스 초기화 블럭 → 인스턴스 초기화 블럭 → 생성자

## this와 this()
`this`는 현재 인스턴스 자기 자신을 참조하는 키워드로, 주로 지역변수와 인스턴스 변수의 이름이 겹칠 때 `this.변수명` 형태로 구분하기 위해 사용한다.
`this()`는 같은 클래스 내의 다른 생성자를 호출할 때 사용하며, 반드시 생성자의 첫 줄에 위치해야 한다.

## 오버로딩(Overloading)
오버로딩은 하나의 클래스 내에서 같은 이름의 메서드를 여러 개 정의하는 것으로, 매개변수의 타입이나 개수가 달라야 한다.
반환 타입은 오버로딩의 조건에 해당하지 않는다. 
같은 기능을 수행하지만 입력값의 형태가 다를 때 메서드의 이름을 통일하여 사용할 수 있어 코드의 가독성을 높여준다.
