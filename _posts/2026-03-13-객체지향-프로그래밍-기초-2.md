## 객체지향의 특징 
객체지향 프로그래밍에는 캡슐화, 상속, 다형성, 추상화라는 네 가지 핵심 특징이 있다.
- **캡슐화(Encapsulation)**: 데이터와 메서드를 하나로 묶고, 외부에서 직접적인 접근을 제한하여 보호
- **상식(Inheritance)**: 부모의 특성과 기능을 자식이 그대로 물려받아 코드의 재사용성을 높임
- **다형성(Polymorphism)**: 하나의 참조변수 타입으로 여러 타입의 객체를 참조하는 성질을 의미
- **추상화(Abstraction)**: 객체의 핵심적인 공통 속성과 기능만을 추출하는 것

## 캡슐화
자바에서는 제어자를 통해 캡슐화를 구현할 수 있다. 제어자(modifier)는 클래스, 변수, 메서드 선언부에 함께 사용되며 접근 범위를 제어하는 **접근 제어자**와 사용 방식을 제한하는 **기타 제어자**가 있다.
접근 제어자는 하나의 대상에 하나만 적용 가능하지만 기타 제어자는 여러 조합이 가능하다. 

### 접근 제어자
말 그대로 접근을 제어한다. 클래스, 멤버 변수, 메서드, 생성자에 사용 가능하다. 
변수를 선언할 때 `private`를 사용하면 해당 클래스 내에서만 사용할 수 있으므로 외부에서 함부로 값에 접근하거나 변경하는 것을 방지할 수 있다. 

| 제어자       | 같은 클래스 | 같은 패키지 | 자식 클래스 | 전체 |
|-----------|--------|--------|--------|----|
| public    | ✅      | ✅      | ✅      | ✅  |
| protected | ✅      | ✅      | ✅      | ❌  |
| (default) | ✅      | ✅      | ❌      | ❌  |
| private   | ✅      | ❌      | ❌      | ❌  |

### 기타 제어자
객체의 사용 방식이나 특성을 결정한다. 주로 사용되는 것은 `static`, `final`, `abstract` 이다. <br/>

`static`은 '클래스의', '공통적인'의 의미를 가지며, 멤버변수, 메서드, 초기화 블럭에 사용된다.
`static`이 붙은 멤버는 클래스가 메모리에 로드될 때 생성되어 프로그램 종료까지 유지되며, 인스턴스 생성 없이도 사용할 수 있다.
또한 모든 인스턴스가 동일한 값을 공유한다는 특징이 있다. <br/>

`final`은 '마지막의', '변경될 수 없는'의 의미를 가지며, 클래스, 멤버 변수, 메서드 모두에 사용할 수 있다. 
변수에 사용하면 값을 변경할 수 없는 상수가 되고, 메서드에 사용하면 오버라이딩을 통한 재정의가 금지되며, 클래스에 사용하면 상속이 불가능해진다. 
상수는 보통 선언과 초기화를 동시에 하지만, 생성자를 통해 인스턴스별로 단 한번 초기화하는 것도 가능하며 유연한 상수 관리가 가능하다. <br/>

`abstract`은 '추상의', '미완성의' 의미를 가지며, 추상 메서드 또는 추상 메서드를 포함한 클래스임을 명시할 때 사용한다.

> 그 외에는 native, transient, synchronized, volatile, strictfp 이 존재한다.

## 상속
상속은 기존의 클래스를 재사용하여 새로운 클래스를 작성하는 것으로, 클래스는 상속을 거듭할수록 그 내용이 확장되고 구체화된다.
이러한 특성 때문에 '확장한다'는 의미인 `extends` 키워드를 사용한다. <br/>

상속 관계에서는 부모클래스가 변경되면 자식 클래스에 영향을 미치지만, 반대로 자식 클래스가 변경되었을 때는 부모 클래스에 아무런 영향을 주지 않는다.
이는 자식 클래스가 부모 클래스의 멤버를 포함하기 때문이며, 그로 인해 부모 클래스의 멤버 수는 항상 자식 클래스의 멤버 수보다 작거나 같다. <br/>

단, 모든 것이 상속되는 것은 아니다. 생성자와 초기화 블럭은 클래스 고유의 로직이므로 상속되지 않으며 `super()`를 통한 호출만 가능하고,
`static` 멤버 역시 클래스 자체에 귀속되므로 상속대상에서 제외된다. 
또한 자바는 단일 상속만을 지원하는데, 이는 두 부모로부터 동일한 이름의 기능을 물려받을 때 발생할 수 있는 충돌을 방지하기 위함이다. <br/>

한편 부모로부터 물려받은 메서드를 자식에서 재정의하는 행위를 '**오버라이딩**'이라고 부른다. 
오버라이딩은 메서드의 선언부와 반환타입은 부모와 일치하지만, 구현부는 자식에 맞게 수정할 수 있다. 

> 상속을 통해서 우리는 코드의 재사용성을 높이고 중복을 제거하며 생산성과 유지보수성을 향상 시킬 수 있다.
> 단순히 기능을 가져오기 위해 사용하는 것이 아니라 클래스 간의 연관성을 고려하여 상속해야 한다. 

> - **오버라이딩**: 부모로부터 상속받은 메서드를 자식이 재정의하는 것으로, 선언부와 반환 타입은 부모와 동일해야 한다.
> - **오버로딩**: 같은 클래스 내에서 동일한 이름의 메서드를 여러 개 정의하는 것으로, 매개변수의 타입이나 개수가 달라야 한다. 반환 타입은 구분 기준에 해당하지 않는다. 
 
### Super와 Super()
부모의 멤버와 자식 멤버의 이름이 같을 때 이를 구분하기 위해서는 `super.변수명`을 사용하고, 부모의 생성자를 호출할 때는 `super()`를 사용한다.
이때 부모의 생성자는 반드시 자식보다 먼저 호출되어야 한다. 

### 포함관계
상속 외에도 클래스를 재사용하는 방법으로, 클래스의 멤버 변수로 다른 클래스의 참조 변수를 선언하는 방식인 포함관계가 있다. 
어떤 관계를 사용할지는 `is-a(~은 ~이다)`와 `has-a(~은 ~을 가지고 있다)`를 기준으로 구분할 수 있으며, 전자는 상속 관계, 후자는 포함 관계에 해당한다.

## 다형성
하나의 참조변수 타입으로 여러 타입의 객체를 참조하는 성질을 의미한다. 즉, 부모 클래스 타입의 참조 변수로 자식 클래스의 인스턴스를 참조할 수 있다는 것이다.
단, 반대로 자식 타입의 참조변수로 부모 인스턴스를 참조하는 것은 불가능하다. <br/>

이러한 다형성을 활용하면 하나의 부모 타입으로 여러 종류의 자식 객체를 묶어서 관리하거나, 메서드 매개변수에 부모 타입을 사용하여 다양한 자식 객체들을 한 번에 처리할 수 있다. <br/>

다형성은 형변환을 통해 사용할 수 있는 멤버의 범위를 조절할 수 있는데, 이를 **업캐스팅(Up-casting)**과 **다운캐스팅(Down-casting)**이라고 부르며 상속 관계에 있는 클래스 사이에서만 가능하다.
업캐스팅은 자식 인스턴스를 부모 타입으로 형변환하는 것으로 명시적으로 선언하지 않아도 되지만, 다운 캐스팅은 업캐스팅된 인스턴스를 다시 자식 타입으로 되돌리는 것으로 반드시 명시적으로 선언해야 한다.
다운캐스팅은 주로 업캐스팅하여 사용하던 자식 중 자식만의 고유한 기능이 필요할 때 사용하며, 잘못된 형변환으로 인한 런타임 오류를 방지하기 위해 `instanceof` 연산자로 타입을 확인한 후 진행하는 것이 안전하다. <br/>

한편 형변환 시 멤버 변수와 메서드는 서로 다른 방식으로 동작한다. 
멤버 변수는 정적 바인딩으로 참조 변수의 타입에 정의된 변수가 사용되는 반면,
메서드는 동적 바인딩으로 실제 인스턴스에서 오버라이딩된 메서드가 호출된다. 

```java
class Tv {
  boolean power;
  String name = "Parent_TV";

  void power() { power = !power; }

  void showDisplay() {
    System.out.println("기본 TV 화면을 출력합니다.");
  }
}

class CaptionTv extends Tv {
  String text = "Child_Caption";

  @Override
  void showDisplay() {
    System.out.println("자막이 포함된 화면을 출력합니다.");
  }

  void caption() {
    System.out.println("자막 기능 작동 중: " + text);
  }
}

class SmartTv extends Tv {
  @Override
  void showDisplay() {
    System.out.println("스마트 TV의 앱 화면을 출력합니다.");
  }
}

class Main {
  // 매개변수의 다형성: 어떤 TV 자식들이 들어와도 한 번에 처리 가능
  static void repair(Tv t) {
    System.out.println(t.toString() + " 수리 중...");
  }

  public static void main(String[] args) {

    // 1. ✅ 조상 타입의 참조변수로 자손 인스턴스를 참조 (업캐스팅)
    // CaptionTv 인스턴스 멤버 중 Tv로부터 상속받은 멤버만 사용할 수 있다.
    Tv t = new CaptionTv();
    t.power();

    // [동적 바인딩] 변수 t는 Tv 타입이지만, 실제 객체인 CaptionTv의 메서드 호출
    t.showDisplay(); // 출력: 자막이 포함된 화면을 출력합니다.

    // [정적 바인딩] 변수는 참조 변수의 타입을 따름
    System.out.println(t.name); // 출력: Parent_TV

    // 2. ✅ 부모에서 자식으로 다운캐스팅
    // 자식 클래스에만 있는 고유 기능(caption)을 사용하기 위해 다시 자식 타입으로 변환
    if (t instanceof CaptionTv) {
      CaptionTv c = (CaptionTv)t;
      c.caption(); // 자식만의 메서드 호출 가능
    }

    // 3. ✅ 매개변수의 다형성 활용
    repair(new CaptionTv());
    repair(new SmartTv());

    // 4. ✅ 배열을 통한 일괄 관리
    Tv[] tvStorage = { new CaptionTv(), new SmartTv(), new Tv() };
    for (Tv device : tvStorage) {
      device.showDisplay(); // 각 인스턴스에 맞는 화면 출력
    }

    // 5. ❌ 오류 케이스
    // CaptionTv c = new Tv();
    // 참조 변수가 사용할 수 있는 멤버의 개수는 실제 인스턴스의 멤버 개수보다 같거나 적어야 함
    // CaptionTv는 Tv 인스턴스보다 더 많은 멤버를 가지고 있으므로 불가
  }
}
```

## 추상화
객체의 핵심적인 공통 속성과 기능만을 추출하는 것으로, 자바에서는 추상 클래스와 인터페이스를 통해 추상화를 구현할 수 있다. <br/>

**추상 클래스**는 추상 메서드를 하나 이상 포함하는 클래스로, 인스턴스를 직접 생성할 수는 없지만 멤버 변수와 생성자를 가질 수 있다.
클래스와 메서드에 `abstract` 키워드를 붙여 선언하며, 자식 클래스는 상속을 통해 부모의 모든 추상 메서드를 구현해야 한다. <br/>

**인터페이스**는 추상 클래스보다 추상화 정도가 높으며, 기본적으로 추상 메서드와 상수만을 가진다. 
모든 멤버 변수는 `public static final`, 모든 메서드는 `public abstract`로 선언되며 이는 생략 가능하다. 
인터페이스는 인터페이스끼리만 상속받을 수 있으며, 클래스에서 인터페이스를 사용할 때는 구현한다는 뜻의 `implements` 키워드를 사용한다
 
> 디폴트 메서드와 static 메서드 <br/>
> Java8+부터 인터페이스에서도 구현부가 있는 메서드를 추가할 수 있게 되었다. 
> - **static 메서드**: 인터페이스 이름으로 직접 호출하여 사용할 수 있는 유틸리티 성격의 메서드
> - **default 메서드**: 인터페이스에 새로운 메서드를 추가하더라도 기존에 이 인터페이스를 구현했던 클래스를 수정하지 않아도 되도록 하위 호환성을 제공한다. 구현체에서는 필요한 경우에만 선택적으로 오버라이딩하여 사용할 수 있다.
