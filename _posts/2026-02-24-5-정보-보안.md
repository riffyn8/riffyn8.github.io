### 소프트웨어 보안 개발 방법론
1. **MS-SDL**: MS에서 자체적으로 수립한 소프트웨어 개발 모델
2. **Seven Touchpoints**: 실무적으로 검증된 소프트웨어 보안의 모범 사례 7가지를 개발 모델에 통합한 것
3. **CLASP**: 소프트웨어 생명 주기(SDLC) 전반에 걸친 포괄적으로 효율적인 보안 모델
4. **CWE(Common Weakness Enumeration)**: 소프트웨어 보안 취약점을 체계적으로 분류한 DB

### 소프트웨어 개발 보안 요소
- **기밀성(Confidentiality)**: 인가된 사용자만 정보에 접근
- **무결성(Integrity)**: 인가된 사용자만 정보를 수정 가능
- **가용성(Availability)**: 인가된 사용자는 원하는 시점에 서비스 이용 가능
- **인증(Authentication)**: 사용자가 정당한 사용자인지 확인하는 과정
- **부인 방지(Non-Repudiation)**: 데이터를 송수신한 후, 보냈거나 받았다는 사실을 나중에 증명하여 사실 자체를 부인하지 못하게 하는 것

### 사용자 인증
- 지식 기반 인증(Something You Know, 알고 있는 것): 기억하고 있는 정보를 이용해 인증 수행
- 소유 기반 인증(Something You Have, 가지고 있는 것): 소지품을 통해 인증 수행(출입카드, OTP)
- 존재 기반 인증(Something You Are, 스스로의 모습): 생체 정보를 통해 인증 수행
- 행위 기반 인증(Something You Do, 행동하는 것): 행위자의 움직임을 통해 인증 수행
- 위치 기반 인증(Somewhere You Are, 수행하는 곳): 행위자가 자리한 곳의 물리적/논리적 위치를 통해 인증 수행

### 접근 통제 기술
- **임의 접근 통제(DAC)**: 사용자의 신원에 따라 접근 권한을 부여
- **강제 접근 통제(MAC)**: 주체와 객체의 등급을 비교하여 접근 권한을 부여
- **역할 기반 접근 통제(RBAC)**: 사용자의 역할에 따라 접근 권한을 부여

### 접근 통제 모델
- **벨-라파듈라 모델**: 기밀성 강조. 보안 수준이 높은 곳에서 낮은 곳으로 정보가 흐르는 것을 방지(No Read Up, No Write Down, 군대 보안 모델)
- **비바 모델**: 무결성 강조. 보안 수준이 낮은 곳에서 높은 곳으로 정보가 흐르는 것을 방지(No Read Down, No Write Up)
- **클락-윌슨 모델**: 무결성 강조. 사용자가 직접 접근하는 대신 인가된 프로그램을 통해서만 데이터 수정을 허용
- **만리장성(Chinese Wall) 모델**: 무결성 강조. 이해충돌 방지. 서로 상충되는 관계에 있는 객체 간의 정보 흐름을 차단
- **접근 제어 행렬(ACM)**: 주체(행)와 객체(열)의 권한 관계를 2차원 표 형태로 나타낸 가장 기본적인 모델

### AAA
- **Authentication(인증)**: 신원 확인 
- **Authorization(인가)**: 권한 부여 (DAC, MAC, RBAC)
- **Accounting(계정/과금)**: 자원 사용 추적

### 비밀키(대칭키) 암호화
암호화와 복호화에 사용하는 키가 동일한 방식
- 알고리즘이 단순하고 속도가 빠르지만 관리해야 할 키의 개수가 비교적 많은 편
- 필요한 키의 수: $n(n-1)/2$

### 블록 암호
데이터를 특정 크기의 블록 단위로 나누어 암호화
- **DES**: 미국 NBS에서 발표. 64비트 블록과 56비트 키를 사용하며 현재는 사용되지 않음.
- **3DES**: DES를 3중으로 하여 보안성을 강화한 것
- **AES**: DES의 보안 취약성을 해결하기 위해 만든 것으로, 128비트 블록 크기를 가지며 키 길이에 따라 128, 192, 256비트로 나뉜다
- **SEED**: KISA가 개발한 국내 표준 알고리즘
- **ARIA**: 국정원과 학계가 공동 개발한 국내 표준. AES와 호환성이 좋음
- **HIGHT**: 저사양·경량 환경에서 사용하기 위해 KISA 등이 개발한 64비트 블록 암호 알고리즘
- **IDEA**: 스위스에서 개발된 알고리즘. DES보다 안전하고 강력한 성능을 보여줌
- **Skipjack**: 미국 NSA에서 개발한 음성 도청 방지용 알고리즘.
- **RC5**: 비블록 크기, 키 길이를 가변적으로 설정할 수 있으며 속도가 빠르고 단순한 구조를 가진 대칭키 블록 암호 알고리즘

### 스트림 암호
데이터를 비트나 바이트 단위로 흐름(스트림)에 따라 순차적으로 암호
- **RC4**: 가변 키 크기의 스트림 암호로, 구조가 단순하고 속도가 매울 빨라 WEP나 SSL/TLS에 널리 사용되었음
- **LFSR**: 선형 피드백을 이용해 의사 난수를 생성하는 하드웨어 기반 알고리즘. 회로 구성이 간단하여 군용 통신이나 무선 기기에 주로 사용
- **SEAL**: 하드웨어가 아닌 소프트웨어 실행 속도를 최적화하여 설계된 스트림 암호.

### 공개키(비대칭키) 암호화
암호화할 때 쓰는 키(공개키)와 복호화할 때 쓰는 키(개인키)가 서로 다른 방식
- 알고리즘이 복잡하고 속도가 느리지만 관리해야 할 키의 수가 비교적 적은 편
- 키 분배(대칭키 전달용), 전자서명, 부인 방지에 사용된다.
- 필요한 키의 수: $2n$

### 공개키 암호화 종류
- **RSA**: 대표적인 공개키 암호화 알고리즘. 아주 큰 수를 <u>소인수 분해</u>하기 어렵다는 원리를 이용
- **ECC**: 타원 곡선 이산 대수 문제를 이용하여, RSA보다 짧은 키 길이로도 동일한 보안 강도를 제공
- **Diffie-Hellman (디피-헬먼)**: 암호화 목적보다는 키 교화늘 위해 고안되었으며, 이산 대수 계산의 어려움을 이용한 최초의 공개키 알고리즘
- **ElGamal (엘가말)**: 이산 대수의 원리를 이용하여, 암호화할 때마다 암호문이 달라지는 특징이 있어 전자서명과 암호와에 모두 사용됨
- **DSA**: 미국 정부(NIST)에서 표준(DSS)으로 채택한 이산대수 문제 기반(ElGamal 알고리즘 기반)의 전자서명 전용 알고리즘

### 해시 암호
키 없이 단방향(암호화)으로만 변환 가능한 암호화 알고리즘.
- **솔트(Salt)**: 같은 입력에 대해 다른 출력을 얻기 위해 추가하는 무작위 문자열
- **대표적인 알고리즘:**
  - **MD5**: 데이터 무결성을 점검하기 위한 해시 알고리즘. MD4의 확장판으로 보안성이 향상됨
  - **MD4**: 32bit 컴퓨터에 최적화된 해시 알고리즘
  - **SHA**: MD를 대체하기 위해 미국 NIST에서 개발한 해시 암호화 알고리즘

### WEP
유선 LAN에서 기대할 수 있는 수준의 무선 LAN 보안 프로토콜

### TKIP
IEEE 802.11 무선랜 보안에 사용된 웹 방식을 보완한 데이터 보안 프로토콜. 이를 사용한 것을 WPA라고 한다.
- **WPA**: WEB을 보완하기 위해 설계된 무선랜 보안 표준으로, 키가 주기적으로 변경되는 TKIP 암호화 방식을 사용하는 기술

> WIPI는 한국형 무선 인터넷 플랫폼 표준. WPA는 Access 보안을 담당하고, WIPI는 상호 운용성을 담당

### VPN
공용 네트워크를 사설 네트워크처럼 사용할 수 있도록 제공하는 기술. 사용자가 마치 자신의 전용 회선을 사용하는 것처럼 해주는 보안 솔루션
- **SSL VPN**: OSI 4계층 이상에서 암호화 ,구축이 간편
- **IPSec VPN**: OSI 3계층에서 암호화, 보안성이 높음

### 방화벽
불법적인 외부 침입을 차단하여 내부 네트워크를 보호하는 시스템
- 내부 → 외부 (바로 통과) 
- 외부 → 내부 (확인 후 통과)

### 침입 방지 시스템(IPS)
불법적인 외부 침입을 차단하여 내부 네트워크를 보호하는 시스템

### 침입 탐지 시스템(IDS)
네트워크 상의 비정상적인 접근을 탐지하는 시스템
- **HIDS**: 개별 서버나 PC에 직접 설치
- **NIDS**: 네트워크 길목(스위치, 게이트웨이 등)에 설치 (대표적으로 Snort)
- **오탐(False Positive)**: 정상 → 비정상으로 잘못 판단
- **미탐(False Negative)**: 비정상 → 정상으로 잘못 판단

### DMZ(비무장지대)
외부 인터넷과 내부 사설망 사이에 위치한 중간 구역. 외부에서 접근해야 하는 서버들을 내부망과 분리하여, 외부 공격자가 서버를 해킹하더라도 내부망까지는 들어오지 못하게 차단

### SIEM
다양한 장비의 로그를 수집·분석하여 보안 위협을 통합 모니터링

### SSO(Single Sign-On)
단 한 번의 로그인만으로 기업의 각종 시스템을 모두 이용할 수 있게 하는 응용 솔루션

### 세션
사용자가 시스템에 로그인한 순간부터 로그아웃할 때까지의 구간. 세션이 유지되는 동안에는 인증이 필요 없음.

### 세션 하이재킹(Session Hijacking)
세션 인증을 가로채어 중요 자원에 접근하는 공격 방법
- 탐지 방법: 비동기화 상태 탐지, ACK Storm 탐지, 패킷의 유실 탐지, 예상치 못한 접속의 리셋 탐지

### 입력 데이터 검증 및 표현의 보안 약점
- **SQL 삽입(SQL Injection)**: 웹 응용프로그램에 SQL을 삽입하여 DB의 데이터를 유출 및 변조
- **경로 조작 및 자원 삽입**: 입력값에 경로 조작 문자열(예: ../)을 삽입하여, 허용되지 않은 파일이나 자원에 접근하거나 시스템 자원을 수정·삭제하는 보안 약점.
- **XSS**: 웹페이지에 악의적인 스크립트를 삽입하여 정보를 탈취하거나 비정상적인 기능 수행을 유발
- **운영체제 명령어 삽입**: 외부 입력값을 통해 시스템 명령어의 실행을 유도. 권한 탈취 혹은 장애 유발
- **위험한 형식 파일 업로드**: 악성 스크립트 파일을 업로드
- **메모리 버퍼 오버플로**: 메모리 범위ㅈ를 넘어선 위치에서 자료를 읽거나 쓰려고 할 때 발생하는 보안 약점
- **신뢰되지 않는 URL 주소로 자동접속 연결**

### 스택 가드(Stack Guard)
널 포인터 역참조와 같이 주소가 저장되는 스택에서 발생하는 보안 약점을 막는 기술 중 하나

### 접근 제어자
특정 객체를 선언할 때 외부로부터의 접근을 제한하기 위해 사용되는 예약어

| 제어자       | 클래스 내부 | 패키지 내부 | 하위 클래스 | 패키지 외부 |
|-----------|--------|--------|--------|--------|
| Public    | ✅      | ✅      | ✅      | ✅      |
| Protected | ✅      | ✅      | ✅      | ❌      |
| Default   | ✅      | ✅      | ❌      | ❌      |
| Private   | ✅      | ❌      | ❌      | ❌      |

### Cloud HSM
암호 키의 생성, 저장, 관리 및 암호 연산을 수행하는 전용 하드웨어 보안 모듈을 클라우드 환경에서 제공하는 서비스


### 바이러스
복제, 은폐, 파괴 기능을 통해 시스템을 마비시키거나 파일을 파괴하는 악성 코드의 한 종류
- 감염대상(파일)에 기생하며 자신을 끊임없이 복제한다.

### 트로이 목마
정상적인 파일로 가장해 컴퓨터 내부에 숨어 있다가 특정 포트를 열어 공격자의 침입을 도와 정보를 유출

### 웜(Worm)
기생이 아닌 스스로 실행되어 확산된다. 전파속도가 매우 빠르며 시스템에 과부화를 일으켜 마비시킨다

### 스푸핑(Spoofing)
승인된 사용자인 것처럼 속이기 위해 IP나 MAC 주소 등 신뢰 정보를 위조하는 공격
- **ARP Spoofing**: MAC 주소 위조 (2계층)
- **IP Spoofing**: IP 주소 위조 (3계층)
- **DNS Spoofing**: 도메인 이름에 대한 IP 주소를 위조하여 가짜 사이트로 유도

### 스니핑(Sniffing)
네트워크 상에서 통과하는 패킷을 가로채서 몰래 훔쳐봄(도청)
- **스위치 재밍(Switch Jamming)**: 위조된 MAC 주소를 대량으로 전송하여 스위치의 MAC 테이블을 오버플로우 시켜 제기능을 못하게 한 후 스니핑 수행 
  - 스니핑의 사전 작업 
  - Mac Flooding의 한 종류
- **ARP Spoofing**: 가짜 ARP 응답 패킷을 보내, 공격자의 MAC 주소를 게이트웨이의 MAC 주소인 것처럼 속이는 공격 (특정 타겟을 속임)
- **ARP Redirect**: 자신이 게이트웨이인 것처럼 속여 네트워크 내의 모든 호스트가 외부로 나가는 패킷을 공격자에게 보내도록 만드는 공격 (전체 네트워크를 속임)
- **ICMP Redirect**: 라우팅 경로를 속여서 패킷을 가로챔

> 스니핑(도청)을 하기 위해서 스푸핑(속이는 행위)를 한다

### 서비스 거부 공격(Dos)
시스템의 자원을 고갈시켜 정상적인 사용자가 서비스를 이용하지 못하게 만드는 공격
- **죽음의 핑(Ping of Death)**:<u> 정상크기 이상</u>의 거대한 ICMP 패킷을 전송하여 수신 측에서 재조립할 때 버퍼 오버플로우를 일으켜 시스템을 다운시키는 공격
- **SYN Flooding**: TCP의 3-Way-Handshake 취약점을 이용한 공격. 다량의 SNY 패킷을 보내 접속 대기 공간(Backlog Queue)을 가득채워 마비시키는 공격
- **DDoS**: 분산된 다수의 좀비 PC를 이용하여 공격 대상 시스템의 서비스를 마비시키는 공격 방식. DoS는 시스템의 취약점을 이용하는 것이지만, DDoS는 다수의 선량한 PC를 동원함
- **LAND 공격**: 출발지와 목적지 IP를 서버 자신의 주소로 변경하여, 패킷이 자기 자신에게 무한 루프를 돌게 해 자원을 고갈시키는 공격
- **반사 공격**: 출발지 IP를 희생자 주소로 변조하여 제3의 서버에 요청을 보내고, 그 응답이 희생자에게 쏟아지게 하는 공격
- **Smurf**: 출발지 IP를 희생자로 변조하여 브로드캐스트 주소로 ICMP 패킷을 보내, 네트워크 안의 수많은 PC가 동시에 희생자를 공격하게 만드는 증폭 공격
- **ICMP Flooding**: <u>모든 종류의 ICMP 패킷을 대량으로</u> 보내 네트워크를 마비
- **Ping Flooding**: <u>특정 타겟에게 ICMP Echo Request(Ping)을 엄청난 양</u>으로 퍼부어 네트워크 대역폭을 고갈
- **Teardrop**: 패킷 재조립 시 사용되는 오프셋 값을 조작하여 패킷이 중복되거나 빈 공간이 생기게 함으로써 수신 측 시스템을 마비시키는 공격
- **Slowloris**: HTTP 헤더를 아주 천천히 전송하여 웹 서버의 연결 자원을 모두 점유하고, 다른 사용자의 접속을 차단하는 서비스 거부 공격
- **RUDY**: HTTP POST 방식의 Content-Length를 크게 설정한 뒤, 데이터를 1바이트씩 매우 느리게 전송하여 서버 자원을 고갈

> SYN Flooding은 IP 스푸핑이 선행된다. (즉, IP를 속이는 것이 먼저 진행됨)

### 블루투스 보안 공격
- **블루스나핑(BlueSnarfing)**: 블루투스의 취약점을 이용하여 목표 장비의 임의 파일에 접근하는 공격 방법
- **블루버깅(BlueBugging)**: 블루투스 장비 간의 취약한 연결 관리를 악용한 공격
- **블루재킹(BlueJacking)**: 블루투스를 이용해 스팸처럼 명함 등을 익명으로 전송하는 공격 방법
- **블루프린팅(BluePrinting)**: 블루투스 공격 장치의 검색 활동을 의미

### 기타 보안 공격 및 용어
- **제로데이 공격**: 특정 취얌저에 대한 보안 패치나 대응법이 발표되기 전에 해당 취약점을 이용하여 위협을 가하는 공격 방법
- **루트킷**: 시스템 침입 후 침입 사실을 숨기고, 관리자 권한을 유지하기 위해 사용되는 악성 프로그램의 모음
- **백도어**: 시스템 설계 시 편의나 유지보수를 위해 만들어둔 비정상적인 통로를 공격자가 권한 획득 등을 위해 악용하는 트랩 도어
- **세션 하이재킹**: 세션 인증을 가로채어 중요 자원에 접근하는 공격 방법
- **Watering Hole**: 특정 집단이 주로 방문하는 웹사이트를 감염시키고 피해 대상이 그 웹사이트를 방문할 때까지 기다리는 웹 기반 공격
- **Dark Data**: 정보를 수집한 후, 저장만 하고 분석에 활용하고 있지 않은 다량의 데이터
- **Typosquatting**: 사이트 주소를 잘못 입력하는 실수를 이용하여 불법 사이트에 접속하게 되어 피해를 주는 공격 방법
- **Smishing**: SMS 를 이용해 이루어지는 피싱
- **Parming**: 가짜 사이트로 접속을 유도하여 개인 정보를 탈취
- **Spear Phishing**: 특정 그룹을 목표로 진행되는 피싱
- **Baiting**: 무료 영화/웹툰 사이트 등으로 유도하여 진행하는 피싱
- **Quishing**: QR 코드를 통해 가짜 사이트 접속이나 앱 설치를 유도하여 개인 정보를 탈취
- **무차별 대입 공격(Brute Force Attack)**: 패스워드를 찾기 위해 가능성 있는 모든 값을 전부 대입
- **크랙(Crack)**: 암호가 걸려 있는 정품 소프트웨어의 암호화를 풀어 불법적으로 무제한 사용
- **스캐닝**: 네트워크 상의 컴퓨터와 가동 서비스를 탐색하여 장비 구성, 포트 구성 등을 파악
- **랜섬웨어**: 사용자의 컴퓨터에 침입하여 파일을 암호화한 뒤에 해독을 조건으로 돈을 요구
- **키로거**: 사용자의 키보드 움직임을 탐지해 개인 정보 탈취
- **스파이웨어**: 사용자 동의 없이 설치되어 컴퓨터의 정보를 수집
- **CSRF**: 사용자가 자신의 의지와 무관하게 공격자가 의도한 행위를 특정 웹사이트에 요청하게 만드는 공격
- **Open Redirect**: 입력된 외부 URL 주소로 사용자를 강제로 이동(리다이렉트) 시켜, 피싱 사이트 등으로 유도하여 개인정보 탈취
