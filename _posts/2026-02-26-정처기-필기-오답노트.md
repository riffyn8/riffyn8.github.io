### 하향식 테스트 방법과 상향식 테스트 방법 비교

| 구분    | 하향식(Top-Down)      | 상향식(Bottom-Up)     |
|-------|--------------------|--------------------|
| 방향    | 상위 → 하위            | 하위 → 상위            |
| 도구    | 스텁(가짜 하위 모듈)       | 드라이버(테스트 수행 상위 모듈) |
| 인터페이스 | 상위에서 하위로 내려가며 정의 됨 | 미리 정의해두어야 함.       |
| 장점    | 주요 설계 결함을 빨리 발견    | 하위 모듈의 정밀한 테스트 가능  |

- 하향식 테스트에서는 통합 검사 시 인터페이스가 이미 정의되어 있어 통합이 간단
- 하향식 테스트에서 레벨이 낮은 데이터 구조의 세부 사항은 테스트 초기 단계에서 필요
- 상향식 테스트는 최하위 수준에서 각각의 모듈들을 테스트하고 이러한 모듈이 완성되면 이들을 결합하여 검사 

### 아키텍처 설계 단계
`설계 목표 설정 → 시스템 타입 설정 → 아키텍처 패턴 적용 → 서브 시스템 구체화 → 검토`

### FEP

### 정형 기술 검토(FTR)
개발 과정에서 만들어진 결과물에 결함이 있는지 사람들이 모여서 검토하는 활동
- 제품의 검토에만 집중
- 의제를 제한
- 논쟁과 반박을 제한
- 문제 영역을 명확히
- 의견을 서면으로 작성
- 참가 인원을 제한
- 사전 준비를 강요
- 자원과 시간 일정을 할당
- 모든 검토자들을 위해 적절한 교육을 행하기
- 검토 과정과 결과를 재검토

### ISO/IEC 25000 (SQuaRE)
소프트웨어 제품의 품질 요구사항 정의 및 품질 평가를 위한 국제 표준

| 2501n | 품질 모델   | 품질 특성(기능성, 신뢰성 등) 정의 (가장 핵심) |

### 서브넷 문제1
10.100.240.0/22 네트워크에서 FLSM 방식으로 10개의 서브넷으로 나누었을 때, 2번째 서브넷의 5번째 유효한 IP 주소는 무엇인가? (ip subnet-zero는 적용하지 않는다.)

- **✔️ 10.100.240.133**

> 1. 10개의 서브넷을 만들기 위한 최소한의 비트($10>=2^n$)를 구해야함 → $2^4=16$
> 2. 기존 네트워크가 `/22`이므로, 서브넷을 나누기 위해 호스트 비트에서 4비트를 더 빌려온다 → `/22 + 4 = /26` 
> 3. 서브넷당 호스트 비트는 %32-26=6%. 6비트이므로, 서브넷당 IP 개수는 $2^6=64$개가 된다.
> 4. 0번은 10.100.240.0 ~ 63, 1번은 10.100.240.64 ~ 127, 2번은 10.100.240.128 ~ 191. 여기서 ip subnet-zero를 적용하지 않았으므로 0번 서브넷은 제외.
> 5. 실질적으로 2번 서브넷이 2번째 서브넷이 되고, 여기서 5번째는 10.100.240.133이 된다.

### MIME
전자우편 서비스에서 멀티미디어 메일을 주고받기 위한 프로토콜

### 상위/하위 설계 단계
- 상위 설계(아키텍처, 예비 설계): 시스템 구조, DB 설계, 화면 레이아웃
- 하위 설계(모듈, 상세 설계): 구성 요소들의 내부 구조(로직), 동적 행위(절차) 등

### 소프트웨어 유지보수 유형
- 하자 보수(버그 수정), 완전 보수(성능 개선), 예방 보수(문제 발생 방지), 적응 보수(환경 변화 대응) 

### 인스펙션
