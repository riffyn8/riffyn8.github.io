## 네트워크 이론
### 데이터 전송 방식
- **동기화(Synchronization)**: 송신측과 수신측이 동일한 타이밍에 맞춰 데이터를 송수신하는 것
- **동기식 전송(Synchronous)**: 데이터를 블록(프레임) 단위로 전송. 전송 효율 및 속도가 높다
- **비동기식 전송(Asynchronous)**
  - 데이터를 문자 단위로 전송. 문자 마다 시작/정지 비트를 붙인다.
  - 저속 전송에 적합하며, 구조가 단순하고 비용이 저렴

### BSC
제어 문자를 사용하여 프레임을 구성하는 문자 위주의 동기식 전송 제어 프로토콜

### 동기식 전송 제어 문자
- **SYN**: 송수신 측의 타이밍(동기)를 맞추기 위해 사용. 
- **SOH**: 전송되는 프레임의 제어 정보가 시작됨을 알림
- **STX** / **ETX**: 데이터(본문)의 시작과 종료를 알림
- **ACK** / **NAK**: 전송 성공 여부를 알리는 신호로 오류 제어에 사용된다. (긍정, 부정 응답)
- **DLE**: 데이터 내에 제어 문자와 같은 비트열이 포함될 경우, 이를 데이터로 인식하게 만드는 투명성을 제공할 때 사용

### 순방향 오류 수정(FEC)
송신측이 데이터에 오류 수정용 부가 정보를 담아 보내면, 수신측에서 이를 이용해 직접 오류를 발견하고 수정
- 역채널이 필요 없고 재 전송에 따른 지연이 없으나, 부가 정보 때문에 전송 효율이 낮아질 수 있다.
- **해밍코드(Hamming Code)**: 패리티비트를 이용해서 오류를 검출(2bit)하고, 스스로 수정(1bit)하는 방식
- **상승코드**: 데이터를 연속적으로 처리하여 여러 비트의 오류를 스스로 수정

### 역방향 오류 수정(BEC)
수신측에서 오류를 발견하면 역채널을 통해 송신측에 알려서 재전송(ARQ)을 통해 오류를 해결
- 신뢰성이 높으며 가장 많이 사용 된다
- **패리티 검사**: 전송 비트 중 1의 개수를 홀수나 짝수로 맞춰서 보내는 방식 (1bit 에러를 검출)
- **블록합 검사(BSC, LRC)**: 패리티의 단점을 보완해 가로, 세로 양방향으로 검사
- **CRC(순환 중복 검사)**: 다항식을 이용. HDLC 등에서 쓰이는 가장 신뢰성 높은 방식

### ARQ
오류가 발생할 경우 자동으로 송신측에 데이터의 재전송을 요청하는 것.
- **Stop-and-Wait (정지-대기)**: 하나의 프레임을 보내고 ACK을 받을 때까지 다음 프레임을 보내지 않고 기다림
- **Go-Back-N**: 오류가 발생한 프레임부터 그 이후에 전송된 모든 프레임을 폐기하고 전부 다시 재전송
- **Selective Repeat (선택적 재전송)**: 수신 측에서 오류가 발생한 특정 프레임만 골라서 재전송하는 방식이며, 수신 측에 별도의 버퍼가 필요
- **Adaptive (적응형)**: 통신 채널의 효율에 따라 프레임의 길이를 동적으로 변경하여 전송 효율을 극대화하는 방식 

> ARQ는 오류 제어뿐만 아니라 송·수신측의 속도를 맞추는 흐름 제어의 수단으로도 사용 됨.
> - 정지-대기, 슬라이딩 윈도우
> - **슬라이딩 윈도우**: 수신 측이 받을 수 있는 데이터의 크기(Window)를 전달하는 것

### CSMA
회선이 사용 중인지 먼저 살피고, 비어 있을 때 누구나 데이터를 보내는 방식
- **CSMA/CD**: 유선 LAN 방식. 전송 중 충돌이 발생하면 즉시 탐지하고 재전송 (IEEE 802.3)
- **CSMA/CA**: 무선 LAN 방식. 충돌을 막기 위해 회선이 비어도 일단 대기하여 충돌을 회피 (IEEE 802.11)

### IEEE 802 표준 규약
- **802.1**: 전체적인 구성, 관리 및 브리지(Bridge) 규격
- **802.2**: LLC 계층 (데이터 링크 계층의 상위 부분)
- **802.3**: CSMA/CD 방식의 유선 LAN (이더넷)
- **802.4**: 토큰 버스(Token Bus) 방식의 LAN
- **802.5**: 토큰 링(Token Ring) 방식의 LAN
- **802.11**: CSMA/CA 방식의 무선 LAN (Wi-Fi)
- **802.15**: 블루투스(Bluetooth) 등 근거리 개인 무선망(WPAN)

### 네트워크 토폴로지
- **성형(Star, 중앙 집중형)**: 중앙 노드와 1:1로 연결하며, 중앙 노드가 고장나면 전체가 마비됨
- **버스형(Bus)**: 하나의 통신 회선에 다수의 단말기가 연결되어 있음. 단말기 추가/제거가 용이하다. 그러나 통신 회선의 길이가 제한적이고 기밀 보장이 어려움
- **환형(Ring, 루프형)**: 인전합 노드끼리 원형으로 연결하며, 한 노드라도 고장 나면 전체가 마비됨
- **망형(Mesh)**: 모든 노드를 그물처럼 서로 연결하여 신뢰성이 가장 높음. 비용이 많이 듦
- **트리형(Tree, 분산형)**: 계층적 구조로 통신망을 확장하여 분산 처리 시스템에 적합

### 패킷 교환 방식
- **가상 회선**: 패킷 전송 전 경로를 미리 설정하여 모든 패킷이 정해진 순서대로 도착하는 방식 (TCP)
- **데이터그램**: 경로 설정 없이 패킷마다 독립적으로 전송되어 도착 순서가 바뀔 수 있음 (UDP)

### 네트워크 장비
- **허브(Hub)**: 여러 대의 컴퓨터를 연결하여 네트워크를 구성
- **리피터(Repeater)**: 감쇠된 전송 신호를 증폭하여 전송 거리를 연장
- **브리지(Bridge)**: 두 개의 LAN을 연결하여 트래픽을 분산하고 충돌을 방지
- **스위치(Switch)**: 하드웨어 기반으로 MAC 주소를 참조하여 데이터를 목적지에 빠르게 전달
- **라우터(Router)**: 데이터 전송의 최적 경로를 설정하고 서로 다른 네트워크를 연결
- **게이트웨이(Gateway)**: 서로 다른 통신망에 접속하기 위해 사용

### 다중화기(Multiplexer, MUX)
하나의 통신 회선을 여러 단말기가 공유하여 효율을 높이는 장치
- **주파수 분할 다중화(FDM)**: 전송 회선의 대역폭을 다수의 작은 채널로 분할하여 동시에 이용
- **시분할 다중화(TDM)**: 전송 회선의 대역폭을 타임 슬롯으로 나누어 채널에 할당
  - **STDM(동기식 시분할 다중화)**: 채널을 정적으로 분배. 단순하지만 타임 슬롯 낭비 심함
  - **ATDM(비동기식 시분할 다중화)**: 채널을 동적으로 분배. 다양한 기능 지원 필요(비용 상승)
- **코드 분할 다중화(CDM)**: FDB + TDM. TDM으로 전송 신호를 할당하고, 할당된 타임 슬롯에는 FDM으로 할당
- **파장 분할 다중화(WDM)**: 광섬유 등을 매체로 광신호를 전송할 때 사용

### 프로토콜
서로 다른 시스템에 존재하는 노드 간의 원활한 통신을 위한 규칙과 약속
- 구성 요소: 구문(Syntax), 의미(Semantics), 시간(Timings)

### IGP(내부 라우팅)
하나의 자치 시스템 내에서 경로를 주고받는 프로토콜
- **RIP**
  - 벨만-포드 알고리즘 기반 (거리 벡터)
  - 경유하는 라우터의 수(Hop Count)만 따짐.
  - 최대 홉 수를 15로 제한하는 소규모용 프로토콜
- **OSPF**
  - 다익스트라 알고리즘 기반 (링크 상태)
  - 네트워크 변화가 있을 때만 업데이트하여 효율적이다.
  - 계층적 구조로 관리하여 대규모 네트워크에 적합
- **IGRP**: RIP의 홉 수 제한 문제를 해결하기 위해 시스코가 개발
- **EIGRP**: 거리 벡터와 링크 상태의 장점을 합친 하이브리드 방식. 빠른 수렴 속도를 자랑한다.

### EGP(외부 라우팅)
서로 다른 자치 시스템 간에 정보를 교환하는 프로토콜
- **BGP**
  - 경로 벡터 방식 사용
  - 초기 연결 시 전체 테이블을 교환하고, 이후에는 변경된 부분만 전송한다.
  - TCP를 사용해 신뢰성이 높다.
  - 인터넷의 핵심 경로 정보를 교환한다.

### ITU-T X.25 프로토콜
패킷 교환망에서 DTE(사용자 단말기)와 DCE(네트워크 종단 장치) 간의 인터페이스를 제공하는 표준 프로토콜
- 연결형(가상회선) 방식을 사용하여 데이터 전송의 신뢰성을 보장한다.
- 1계층: X.21 (물리적 연결 표준)
- 2계층: LAPB (에러 제어 및 흐름 제어 담당)
- 3계층: PLP (패킷의 구성 및 가상 회선 제어)

### OSI 7계층
1. **물리(Physical) 계층**: 전기적, 기계적 신호를 통해 비트(0, 1) 데이터를 물리적으로 전송
2. **데이터 링크(Data Link) 계층**: 인접 노드 간 프레임 전송, 오류 제어 및 흐름 제어 수행. MAC 주소 통신
3. **네트워크(Network) 계층**: 패킷의 최적 경로를 설정(라우팅) 및 IP 주소 부여(라우터)
4. **전송(Transport) 계층**: 송수신 프로세스 간(End-to-End) 투명하고 신뢰성 있는 데이터 전송을 담당.
5. **세션(Session) 계층**: 송수신측 간의 대화 제어 및 연결 유지/관리
6. **표현(Presentation) 계층**: 데이터 암호화, 압축, 구문 변환 및 코드 변환
7. **응용(Application) 계층**: 사용자 또는 소프트웨어에 네트워크 서비스를 직접 제공

> 데이터 단위(PDU)와 네트워크 장비
> - `물리`: Bit / 허브, 리피터
> - `데이터 링크`: Frame / 브릿지, 스위치
> - `네트워크`: Packet / 라우터
> - `전송:` 세그먼트(TCP), 데이터그램(UDP)
> - `세션~응용`: Message

### TCP/IP
데이터를 응용 프로그램에 맞추어 송수신하기 위한 프로토콜 및 필수 요건으로 4계층으로 구성되어 OSI 7계층 모델을 대체할 수 있다.

#### 응용 계층 (Application Layer)
사용자가 네트워크에 접속하여 서비스를 이용하는 계층 (OSI 5~7 계층)

| 프로토콜명           | 포트     | 설명                               |
|-----------------|--------|----------------------------------|
| **HTTP**        | 80     | 웹 페이지(HTML)을 주고받기 위한 통신 규약       |
| **HTTPS**       | 443    | HTTP에 보안을 강화                     |
| **FTP**         | 20, 21 | 서버와 클라이언트 간 파일 전송 20-데이터 / 21-제어 |
| **SMTP**        | 25     | 사용자의 메일을 메일 서버로 보낼 때 사용          |
| **POP3**        | 110    | 메일 서버의 메일을 내 컴퓨터로 가져오기           |
| **IMAP**        | 143    | 서버에 메일을 두고 여러 장치에서 동기화하며 확인      |
| **DNS**         | 53     | 도메인 이름을 IP 주소로 변환                |
| **SNMP**        | 161    | 네트워크 장비를 감시하고 관리                 |
| **Telnet**      | 23     | 원격 컴퓨터에 접속하여 CLI 기반 제어           |
| **SSH**         | 22     | 텔넷의 보안 취약점을 해결한 암호화 원격 접속        |

#### 전송 계층 (Transport Layer)
양 종단(End-to-End) 간의 신뢰성 있는 데이터 접송 및 흐름제어 담당
- **TCP**: 연결형, 신뢰성 보장, 흐름/혼잡 제어, 3-Way Handshake 사용
- **UDP**: 비연결형, 속도 빠름, 신뢰성 낮음, 실시간 스트리밍에 적합

#### 인터넷 계층 (Internet)
데이터 패킷의 최적의 경로를 설정(라우팅)하고 주소를 지정
- **IP**: 주소를 지정하고 패킷을 목적지까지 전달하는 핵심 프로토콜. 비연결성, 비신뢰성
- **ICMP**: 패킷 전송 중 발생한 오류 보고 및 네트워크 진단 (ping)
- **IGMP**: 호스트와 라우터 간의 멀티캐스트 그룹 유지
- **ARP**: 논리주소(IP)를 물리주소 (MAC)으로 변환
- **RARP**: MAC → IP

#### 네트워크 액세스 계층 (Network Access)
물리적인 주소(MAC)를 사용하여 실제 데이터를 전기적 신호로 전송
- OSI 물리 + 데이터링크 계층
- **Ethernet**: 유선 LAN의 가장 대표적인 표준(IEEE 802.3, CSMA/CD)
- **IEEE 802**: LAN 표준 규격
- **HDLC**: 신뢰성 높은 고속 동기식 전송 프로토콜. 비트 동기 방식
- **PPP**: 직렬 회선 연결, 에러 검출과 인증 제공

### TCP 헤더
![tcp헤더](tcpheader1.png)

| 명칭                                | 설명                                               |
|-----------------------------------|--------------------------------------------------|
| **Source Port (발신지 포트)**          | 송신 측의 프로세스 번호                                    |
| **Destination  Port (목적지 포트)**    | 수신 측의 프로세스 번호                                    |
| **Sequence Number (수신 번호)**       | 송신하는 데이터의 순서를 지정. 수신측에서 쪼개진 패킷을 재조립할 때 사용.       |
| **Acknowledgment Number (응답 번호)** | 수신된 일련 번호에 대한 다음 수신 패킷의 번호 (성공적으로 수신했음을 알림)      |
| **HLEN (헤더 길이)**                  | TCP 헤더 길이                                        |
| **예약된 필드 (Reserved)**             | 제어 기능(Flag)를 추가할 때 사용할 공간. 비어있음                  |
| **Control Bits (Flag Bits)**      | 현재 패킷의 용도를 나타내는 비트(URG, ACK, PSH, RST, SYN, FIN) |
| **Window Size**                   | 수신 측의 여유 버퍼 크기(한 번에 전송할 수 있는 데이터 양, 흐름 제어에 사용)   |
| **Checksum**                      | 헤더와 데이터 전체의 에러 검출을 위한 값                          |
| **Urgent Pointer**                | 긴급 데이터 처리 목적 (URG 플래그 설정 시)                      |

- **URG**: 긴급 데이터 포함 
- **ACK**: 응답 번호 유효함 
- **PSH**: 데이터를 즉시 상위 계층으로 전달 
- **RST**: 연결 강제 리셋 
- **SYN**: 연결 설정 요청 (3-way handshake의 시작)
- **FIN**: 연결 종료 요청

### IP 헤더
![ip헤더](ipheader1.png)

| 명칭                                | 설명                                               |
|-----------------------------------|--------------------------------------------------|
| **Version**                       | IP 프로토콜의 버전                                      |
| **Header Length**                 | IP 헤더의 전체 길이 (32비트 워드 단위)                        |
| **Type of Service (ToS, 서비스 타입)** | 서비스의 우선순위 및 품질(QoS)을 지정                          |
| **Total Length**                  | 헤더와 데이터를  합친 패킷의 전체 길이 (바이트 단위)                  |
| **Identification (식별자)**          | 단편화된 패킷들을 재조합할 때 동일한 데이터임을 식별하기 위해 부여하는 번호       |
| **TTL (Time To Live)**            | IP 패킷의 수명. 패킷이 네트워크상에 무한정 머물지 않도록 제한하는 시간        |
| **Protocol Type**                 | 데이터에 포함되는 상위 프로토콜 종류를 식별 (TCP=6, UDP=17, ICMP=1) |
| **Header Checksum**               | IP 헤더 자체의 오류 검출을 위한 값 (데이터는 체크하지 않음)             |
| **Source IP Address**             | 송신측의 IP 주소 (32비트)                                |
| **Destination IP Address**        | 수신측의 IP 주소 (32비트)                                |

### IPSec
보안에 취약한 구조를 가진 IP를 개선하기 위해 설계한 표준. 인터넷계층(네트워크 계층)에서 작동. IPv6에서는 기본 사양으로 포함되는 네트워크 계층 보안 표준 프로토콜
- **AH**: 인증과 무결성을 보장. 발신처가 확실한지 확인하고 데이터가 변조되지 않았음을 보장하지만, 암호화(기밀성)은 제공하지 않음
- **ESP**: 인증과 무결성뿐만 아니라 암호화를 추가로 제공.
- **IKE**: 보안 정책을 협상하고 암호화에 사용할 키를 교환 및 관리
- **전송 모드**: IP 페이로드(데이터)만 보호, 호스트 간(End-to-End) 통신 사용
- **터널 모드**: IP 패킷 전체(헤더 포함)를 암호화하고 새로운 IP 헤더를 추가. VPN 구축시 주로 사용

### IPv4
전 세계의 인터넷이 가능한 기기에 부여되는 유일한 식별 주소.
- 8비트씩 4부분으로 구성되는 32비트 주소 체계
- 각 자리는 0~255까지의 10진수로 표현하며 점(`.`)으로 구분
- 주소 부족 문제 해결 방안: `CIDR, DHCP, NAT, IPv6`
- **유니캐스트**: 1:1 방식으로 특정 수신자에게만 데이터를 전송
- **멀티캐스트**: N:M 방식으로 특정 그룹 수신자들에게 데이터를 동시 전송
- **브로드캐스트**: 같은 영역(도메인)에 있는 모든 단말기들에게 한 번에 전송

| 클래스   | 설명               | 범위                  |
|-------|------------------|---------------------|
| A 클래스 | 대규모 네트워크 환경에 사용  | 0 ~ 127, $2^{24}$   |
| B 클래스 | 중규모 네트워크 환경에 사용  | 128 ~ 191, $2^{16}$ |
| C 클래스 | 소규모 네트워크 환경에 사용  | 192 ~ 223, $2^8$    |
| D 클래스 | 멀티캐스트용으로 사용      | 224 ~ 239           |
| E 클래스 | 실험적 주소이며 공용되지 않음 | 240 ~ 255           |

### CIDR

![cidr](cidr1.png)

기존의 클래스 단위로 주소를 나누던 방식이 아닌, 비트 단위로 자유롭게 주소 범위를 나누는 방식
- 클래스에 제한 없이 서브넷 마스크를 이용해 IP 주소를 효율적으로 할당
- IP 주소 뒤에 슬래시(`/`)와 비트 수를 표기하며, 표시된 비트는 네트워크 비트(1)를 의미
- 전체 32비트 중 네트워크 비트를 제외한 나머지 비트는 호스트 비트(0)가 된다

### DHCP
- 네트워크에 접속하는 기기에 IP 주소를 자동으로 할당해주는 프로토콜 (응용 계층에서 사용)
- `Discover → Offer → Request → Ack`

### NAT
네트워크 주소 변환. 사설 네트워크에 속한 IP를 공인 IP 주소로 변환

### IPv6
IPv4의 단점을 보완하기 위해 네트워크 기능 확장성을 지원하고, 보안 및 서비스 품질 기능 등이 개선됨
- 주소 자동 설정 기능을 통해 손쉽게 이용자의 단말을 네트워크에 접속시킬 수 있음
- 16비트씩 8부분으로 구성되는 128비트 주소 체계
- 각 자리는 16진수로 표현하며 콜론(`:`)으로 구분
- 기존 IPv4에서 브로드캐스트 대신 애니캐스트가 사용됨
- **애니캐스트**: 수신 가능한 가장 가까운 수신자에게 데이터를 전송하는 방식

### 서브넷 마스크
IP 주소에서 네트워크 부분과 호스트 부분을 구별하기 위해 사용하는 32비트 마스크 
- 네트워크 부분은 1, 호스트 부분은 0
- IP 주소와 서브넷 마스크를 이진수로 AND 연산하면 해당 IP가 속한 네트워크 주소를 추출할 수 있다

| 클래스   | 서브넷 마스크       |
|-------|---------------|
| A 클래스 | 255.0.0.0     |
| B 클래스 | 255.255.0.0   |
| C 클래스 | 255.255.255.0 |


### 서브넷팅
배정받은 하나의 네트워크 주소를 다시 여러 개의 작은 하위 네트워크로 나누어 사용하는 방식
- **FLSM**: 동일한 크기로 서브넷을 나누는 방식
- **VLSM**: 동일하지 않은 크기로 서브넷을 나누는 방식
- 호스트 비트가 $h$개일 때 해당 서브넷의 전체 IP 개수는 $2^h$개 이며, 가용 호스트 수는 $2^h-2$개이다.
- 가용 호스트 수에서 2를 빼는 이유는 네트워크 주소와 브로드캐스트 주소를 제외해야 하기 때문이다.
- $n$개의 서브넷이 필요하다는 것은 $n <= 2^x$ 범위를 찾아야 한다. <br/> ($x$는 네트워크 비트가 되고, 8비트중 남은 비트는 호스트 비트가 된다 )
- **ip subnet-zero**: 서브넷 마스킹으로 인해 생성된 네트워크 주소 중 첫 번째 주소(모든 서브넷 비트가 0인 주소)를 유효한 IP 주소로 사용할 수 있게 허용 

### 서브넷팅 예제 모음

#### 예제1
IP 주소가 139.127.19.132이고, 서브넷마스크가 255.255.255.192일 때 네트워크 주소와 호스트 개수를 구하시오

- **✔️ 네트워크 주소: 139.127.19.128**
- **✔️ 호스트 개수: 62개**

> 네트워크 주소 구하기 <br/>
> 139.127.19.132와 255.255.255.192를 2진수로 변환 후 AND 연산을 하면, 139.127.19.128 <br/>
> 1옥텟부터 3옥텟까지는 AND 연산을 해도 원래 IP 주소인 139.127.19와 동일하므로, 4옥텟 부분만 2진수로 변환하여 연산

> 호스트 개수 구하기 <br/>
> 255.255.255.192의 2진수로 변환했을 때 호스트 비트는 6개 이므로, $2^6-2=62$

#### 예제2
192.168.0.1/25가 의미하는 서브넷 마스크 값은? 

- **✔️ 서브넷 마스크 값: 255.255.255.128**

> /25는 네트워크 주소가 25개, 즉 4옥텟의 2진수 값은 10000000 이므로 이를 10진수로 변환하면 128

#### 예제3
IPv4의 C 클래스 네트워크를 26개의 서브넷으로 나누고, 각 서브넷에서는 4~5개의 호스트를 연결하려고 한다. 이러한 서브넷을 구성하기 위한 서브넷 마스크 값은?

- **✔️ 서브넷 마스크 값: 255.255.255.248**

> C 클래스의 서브넷 마스크는 255.255.255.0이며, 4옥텟의 8비트(0000 0000)를 네트워크와 호스트 영역으로 나누는 문제 
> - 26개의 서브넷을 만들려면 $2^n >= 26$ 을 만족하는 $n$을 찾아야 한다. 
> - $2^5=32$. 5개는 네트워크 비트, 3개는 호스트 비트가 된다. 
> - 11111000을 10진수로 변환하면 248

#### 예제4
B 클래스 주소를 가지고 서브넷 마스크 255.255.255.240으로 서브넷을 만들었을 때 나오는 서브넷의 수와 호스트의 수는?

- **✔️ 서브넷의 수: 4096**
- **✔️ 호스트의 수: 14**

> B 클래스의 주소에서 몇 비트를 빌려왔는지 계산해야 한다. 
> - B 클래스는 255.255.0.0이므로, 3옥텟과 4옥텟을 빌려온 것을 알 수 있다. 
> - 3옥텟과 4옥텟을 2진수로 변환하면 11111111.1111000 으로 총 12비트를 빌림 
> - 이를 2의 거듭제곱하면 서브넷의 수를 구할 수 있다. $2^{12}=4096$
> - 호스트의 수는 $2^4-2=14$ 

#### 예제5
B 클래스에서 6개의 서브넷이 필요할 때, 가장 많은 호스트를 사용할 수 있는 서브넷 마스크 값은? 
- (1) 255.255.192.0
- **✔️(2) 255.255.224.0**
- (3) 255.255.240.0
- (4) 255.255.248.0

> B 클래스의 주소에서 빌려와야하는 비트 중, $2^n >= 6$을 만족하는 가장 작은 비트 값을 찾아야함
> - 1번은 2비트를 빌려옴 $2^2=4$ 
> - 2번은 3비트를 빌려옴 $2^3=8$ 
> - 3번은 4비트를 빌려옴 $2^4=16$ 
> - 4번은 5비트를 빌려옴 $2^5=32$

## 네트워크 용어
### Mesh Network
기존 무선 랜의 한계 극복을 위해 등장. 대규모 디바이스의 네트워크 생성에 최적화된 새로운 방식의 네트워크 기술

### VoIP
컴퓨터 네트워크 상에서 음성 데이터를 IP 데이터 패킷으로 변환하여 일반 데이터 망에서 음성 통화를 가능하게 해주는 기술

### RFID
모든 사물에 전자 태그를 부착하고 무선 통신을 이용하여 최대 10m 내의 사물의 정보 및 주변 상황 정보를 감지하는 센서 기술

### NFC
10cm 이내로 좁은 비접촉식 통신 기술

### WIPI
한국형무선 인터넷 플랫폼

### WAP
무선 인터넷 전송 규약


### Beacon
근거리에 있는 스마트폰을 자동으로 인식하여 필요한 데이터를 전송할 수 있는 무선 통신 장치. 최대 50m 거리에서도 통신 가능

### Foursquare
위치 기반 소셜 네트워크 서비스

### PICONET
여러 개의 독립된 통신 장치가 블루투스 및 UWB 기술을 사용하여 통신망을 형성하는 무선 네트워크 구축 기술

### MQTT
TCP/IP 프로토콜 위에서 동작하는 발행-구독 기반의 메시징 프로토콜

### 클라우드 컴퓨팅
컴퓨터, 휴대폰 기기와 같은 통신 기기를 이용해 언제 어디서나 서비스를 이용할 수 있도록하는 기술
- **SaaS(Software as a Service)**: 인프라 + 운영체제 + 소프트웨어
- **PaaS(Platform as a Service)**: 개발을 위한 하드웨어 및 소프트웨어 제공
- **IaaS(Infrastructure as a Service)**: 인프라만을 제공
- **BaaS**: 블록체인의 기본 인프라를 추상화하여 블록체인 응용기술을 제공하는 서비스

### 소프트웨어 정의 데이터 센터(SDDC)
데이터 센터의 모든 자원을 가상화하여, 소프트웨어만으로 통합 관리 및 자동 제어하는 데이터 센터
- **SDC**: 서버 자원을 가상화
- **SDN**: 네트워크 경로 제어를 소프트웨어로 관리
- **SDS**: 여러 저장 장치를 하나의 가상 풀로 묶어 관리

### Zing
기기를 키오스크에 갖다 대면 원하는 데이터를 바로 가져올 수 있는 기술. 10cm 이내 거리에서 3.5Gbps 속도로 데이터 전송이 가능한 초고속 근접 무선 통신

### 애드훅 네트워크
기존의 기지국이나 액세스 포인트 없이, 단말기끼리 자율적으로 연결하여 구성하는 독립적 무선 네트워크. 군사 네트워크에 사용됨

### DPI
OSI 7 전 계층의 프로토콜과 패킷 내부의 콘텐츠를 파악하며 침입 시도, 해킹 등을 탐지하고 트래픽을 조정하기 위한 패킷 분석 기술

### IBN
네트워크 관리자가 의도한 바를 시스템이 자동으로 파악하여 네트워크 구성 및 운영을 최적화하는 지능형 네트워킹 기술

### VLAN
하나의 물리적 네트워크를 여러 논리적 네트워크로 분할하는 기술

### Oauth
다양한 플랫폼의 특정 사용자 데이터에 접근하기 위해 비밀번호를 제공하지 않고 3자 클라이언트가 사용자의 접근 권한을 위임 받을 수 있는 프로토콜
- 사용자 인증에 사용되는 표준 인증 방법
- 공개 API로 구현한다.
